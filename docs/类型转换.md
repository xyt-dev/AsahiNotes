# 类型转换

## 自动类型转换

> Types can implicitly be coerced to change in certain contexts. These changes are generally just *weakening* of types, largely focused around pointers and lifetimes. They mostly exist to make Rust "just work" in more cases, and are largely harmless.

### 一般自动类型转换详尽列表

1. ​`T->U`​ 如果 `T`​ 是 `U`​ 的[子类型(subtype)](类型转换/Subtyping.md)
2. 传递性：`T1->T3`​ 如果 `T1->T2`​、`T2->T3`
3. ​`&mut T -> &T`​、`*mut T -> *const T`
4. ​`&mut T -> *mut T`​、`&T -> *const T`
    * 结合以上两条和传递性可知：`&mut T -> *const T`
5. ​`&T / &mut T -> &U`​ 如果 `T`​ 实现了特征 `Deref<Target = U>`
6. ​`&mut T -> &mut U`​ 如果 `T`​ 实现了特征 `DerefMut<Target = U>`
7. ​`TyCtor(T) -> TyCtor(U)`​ 如果 `TyCtor(T)`​ 是以下类型构造器(TyCtor)中的一种

   - ​`&T`
   - ​`&mut T`
   - ​`*const T`
   - ​`*mut T`
   - ​`Box<T>`

   且 `U`​ 要能通过 `T`​ 进行 [Unsized coercion](#unsized-coercion) 得到
8. ​`function item -> function pointer`

   > **注:**
   >
   > ```rust
   > fn test2<T>(arg: T) -> T { arg }
   > let f: fn(u32) -> u32 = test2;
   > ```
   >
   > 顺序：先识别 coercion 规则（function item → function pointer），再在规则约束下进行 unification 推断出 `T = u32`，然后单态化，最后执行 coercion。
   > 单态化构造新类型；coercion 在已有具体类型间转换，不做类型构造。
   >
9. ​`no capture closure -> function pointer`
10. ​`! -> Any Type`

#### **Unsized coercion**

1. ​`[T; n] -> [T]`
2. ​`T -> dyn U`​ 如果 `T`​ 实现了 `U + Sized`​ (前提是 `U`​ 为[支持 dyn 的类型](https://doc.rust-lang.org/reference/items/traits.html#dyn-compatibility))
3. ​`dyn T -> dyn U`​ 如果 `U`​ 是 `T` 的一个 supertrait

### Dot 调用时的自动类型转换

Dot 调用是指：对具有 receiver 的方法的调用

#### 流程：

设 value 为 receiver，value 的类型为 T。

1. 编译器尝试调用 `T::method(value)`，此称为“按值”调用。
2. 若未找到可“按值”调用的方法，则编译器尝试自动为 value 添加引用，也就是编译器会尝试调用 `<&T>::foo(value)`​ 和 `<&mut T>::foo(value)`​，此称为“自动引用“调用。注意：自动 `&T`​ 调用的优先级高于自动 `&mut T` 调用，且只有类型优先级相同情况下才会优先调用结构体本体实现。
   e.g.

   ```rust
   struct S();
   impl S {
       fn test(&mut self) {
           println!("Struct method")
       }
   }

   trait A {
       fn test(&mut self) {
           println!("Trait A method")
       }
   }
   impl A for S {}

   trait B {
       fn test(&self) {
           println!("Trait B method")
       }
   }
   impl B for S {}

   trait C: Sized {
       fn test(self) { // function arguments must have a statically known size
           println!("Trait C method")
       }
   }
   impl C for S {}

   fn main() {
       let mut s = S{};
       s.test();
   }
   // 优先级顺序为: Trait C method > Trait B method > Struct method > Trait A method
   ```
3. 若在步骤 1 和 2 中未能进行方法调用，则会自动尝试使用 `Deref::deref`​ 或对应 `DerefMut::deref_mut`​ 进行解引用(**根据目标调用方法的 receiver 引用类型自动判断应该调用** **​`Deref::deref`​**​ **还是** **​`DerefMut::deref_mut`​**​)，然后对解引用操作的返回值(类型为 `&U`​ 或 `&mut U`​ )递归执行步骤 1 和 3  **(注意：不再执行步骤 2)** 。若无法继续解引用，则会自动尝试进行[动态类型大小转换](#20250419181948-41a13xb)。

> <span id="20250419181948-41a13xb" style="display: none;"></span>**注：** Dot 调用应该只支持 `[T; n]`​ 自动转换为 `[T]`​ 的动态大小类型转换，不支持 `T`​ 自动转换为 `dyn T`:
>
> ```rust
> struct S;
> trait A {}
> trait B {
>     fn test(&self) {
>         println!("Trait B method")
>     }
> }
> trait C {
>     fn test(&self) {
>         println!("Trait C method")
>     }
> }
>
> impl A for S {}
> impl B for dyn A {}
> impl C for [S] {}
>
> fn main() {
>     let s1 = [S{}, S{}, S{}];
>     s1.test();
>     let s2 = S{};
>     (&s2 as &dyn A).test(); 
>     // (&s2).test(); // Err: no method named `test` found for reference `&S` in the current scope
> }
> ```

#### 自动类型转换的微妙效果

##### **e.g.1**

```rust
fn do_stuff<T: Clone>(value: &T) {
    let cloned = value.clone();
}
```

​`clone`​ 方法的定义是: `fn clone(&self) -> Self`​，而在这段代码中，`value`​ 的类型是 `&T`​ 且 `T`​ 有特征约束 `Clone`​，所以此处 `value`​ 可以直接作为 receiver "按值"调用 `clone`​ 方法，返回值也就推断为 `T`。

那么如果去掉特征约束 `Clone` ​呢？由于 `T` ​逻辑上未实现 `clone` ​方法，`&T` ​也就不能作为 receiver "按值"调用 `clone` ​方法，所以编译器自动尝试对 `value` ​进行引用调用，此时该方法调用的签名就变为 `fn clone(&&T) -> &T`​ ，因此 Self = `&T`​，编译器发现 `&T`​ 实现了 `Clone`​ 特征，所以此时返回值推断为 `&T`。

##### e.g.2

```rust
#[derive(Clone)]
struct Container<T>(Arc<T>);

fn clone_containers<T>(foo: &Container<u32>, bar: &Container<T>) {
    let foo_cloned = foo.clone();
    let bar_cloned = bar.clone();
}
```

第一眼看上去好像 `Container<T>` ​实现了 `Clone` ​特征，所以 `bar_cloned` ​类型为 `Container<T>`​? 查看官方对 `#[derive(Clone)]` ​的介绍:

> The Clone trait can be used with `#[derive]`​ only if all fields are `Clone`​. The `derived`​ implementation of [`Clone`](https://doc.rust-lang.org/std/clone/trait.Clone.html#tymethod.clone "method std::clone::Clone::clone")​ calls [`clone`](https://doc.rust-lang.org/std/clone/trait.Clone.html#tymethod.clone "method std::clone::Clone::clone") on each field.
>
> **For a generic struct,**   **​`#[derive]`​** ​ **implements** **​`Clone`​**​ **conditionally by adding bound** **​`Clone`​**​ **on generic parameters.**

也就是说，对于泛型结构体，只有当其泛型都绑定了 `Clone`​ 特征约束时，`#derive`​ 宏才会自动给该结构体实现 `Clone`​ 特征。此处 `u32`​ 实现了 `Clone`​ 特征，所以 `Container<u32>`​ 也自动实现 `Clone`​ 特征，于是 `foo_cloned`​ 的实际类型为 `Container<u32>`​，而泛型结构体 `Container<T>`​ 中 `T`​ 没有绑定 `Clone`​ 特征约束，所以原理同 e.g.1，`bar_cloned`​ 实际类型为 `&bar_cloned`。

## 手动强制类型转换

> Casts are a superset of coercions: every coercion can be explicitly invoked via a cast. However some conversions require a cast. While coercions are pervasive and largely harmless, these "true casts" are rare and potentially dangerous. As such, casts must be explicitly invoked using the `as`​ keyword: `expr as Type`.

所有可自动进行的类型转换都可使用 `as`​ 显式地执行，此外 `as` 可用于手动进行强制类型转换的所有正确情况如下：

|Type of `e`|​`U`|Cast performed by `e as U`|
| -----------------------| -----------------------| ----------------------------------|
|Integer or Float type|Integer or Float type|[Numeric cast](类型转换/Numeric%20cast.md)|
|Enumeration|Integer type|[Enum cast](类型转换/Enum%20cast.md)|
|​`bool`​ or `char`|Integer type|[Primitive to integer cast](类型转换/Primitive%20to%20integer%20cast.md)|
|​`u8`|​`char`|[u8 to char cast](类型转换/u8%20to%20char%20cast.md)|
|​`*T`|​`*V`|[Pointer to pointer cast](类型转换/Pointer%20to%20pointer%20cast.md)|
|​`*T where T: Sized`|Integer type|[Pointer to address cast](类型转换/Pointer%20to%20address%20cast.md)|
|Integer type|​`*V where V: Sized`|[Address to pointer cast](类型转换/Address%20to%20pointer%20cast.md)|
|​`&m1 [T; n]`​ / `*m1 [T; n]`|​`*m2 T`|[Array to pointer cast](类型转换/Array%20to%20pointer%20cast.md)|
|Function item|​`*V`​ where `V: Sized`|Function item to pointer cast|
|Function item|Integer|Function item to address cast|
|Function pointer|​`*V`​ where `V: Sized`|Function pointer to pointer cast|
|Function pointer|Integer|Function pointer to address cast|

需要注意的是，转换切片的原始指针时不会调整长度元数据；例如 `*const [u16] as *const [u8]` 会创建一个仅包含原始内存一半的切片。

此外，强制转换不像自动转换一样具有传递性，也就是说即使 `e as U1 as U2`​ 有效，`e as U2` 也不一定有效。

## Transmutes

​`mem::transmute<T, U>`​ 将 `T`​ 的内存强行解释为**任意占用内存大小相同(转换条件)** 的类型 `U`，此外几乎无限制！且在无明确周期情况下转换为引用会产生无限的周期。

```rust
let wtf = unsafe { std::mem::transmute::<u8, bool>(3) };
println!("{}", wtf) // true（Undefined Behavior）
```

​`mem::transmute_copy<T, U>`​ 则更加狂野，将 `T`​ 的内存数据强行按字节从 `&T`​ 中拷贝出前 `size_of::<U>()`​ 个字节然后强制解释为类型 `U`​，使用的唯一条件是 `size_of::<T>() >= size_of::<U>()` 。
